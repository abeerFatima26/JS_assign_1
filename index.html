Difference between HTTP 1.1 and HTTP 2

  • HTTP 1.1 was created in 1997 while HTTP 2 was created much later in 2015
  • HTTP 1.1 is a text based transfer protocol which keeps all requests and responses in plain text format whereas HTTP 2 uses binary framing layer which converts all messages in binary format.
  • In HTTP 1.1 multiple requests are send sequentially requiring multiple connections while HTTP 2 supports multiplexing,which allows multiple requests and response to be sent over a single connection concurrently.
  • HTTP 1.1 compresses data by itself and in HTTP 1.1 header information is sent with each request and response burdening the data bandwidth whereas HTTP 2 uses H-PACK for data compression to reduce the overhead of sending header information with each request and response which results in reduced bandwidth usage and faster and smoother performance
  • Although HTTP 1.1 has multiplexing but still the issue arises when multiple streams awaits the same resource HTTP 2 handles this issue well with the concept of Stream Prioritization. This enables more important resources to be delivered faster, enhancing the user experience.
  • HTTP 1.1 requires opening and closing of connections every time a request is made while in HTTP 2 allows the reuse of single connection for multiple requests and responses.
  • HTTP 2 is designed to be compatible with HTTP 1.1 i.e servers using HTTP 2 can still communicate with clients using HTTP 1.1
  • Overall features like improved performance,multiplexing,header compression and server push makes HTTP 2 a more preferred choice.
